## 4) 구현 매핑 규칙(현재 코드 구조 유지)

- 태그:
  - `BLOCKn_SPEC.files[*].candidates`에서 PREVIEW BUFFER 링크로 노출
- 레시피:
  - 우선 `BLOCKn_SPEC.synthesisRules[]` 매칭
  - 실패 시 태그 분류(`개체/행위/상태/관계/제약`) 기반 일반 합성 규칙 폴백
- 추론 빈칸:
  - `BLOCKn_SPEC.clause.questions[]` 또는 `BLOCKn_SPEC.clause.slots`
- 정답 판정:
  - 슬롯 수용값 `accepts` 일치 시 완료
- 출력:
  - `BLOCKn_SPEC.clause.outputText`에 논리 수식 표기(`:=`, `=>`, `∧`, `¬`) 유지

태그 분류 기반 일반 합성 예:
- `상태 + 개체 -> 상태(개체)`  (예: `비해침 + 인간 -> 비해침(인간)`)
- `행위 + 인자 -> 행위(인자)`  (예: `관측 + 인간 -> 관측(인간)`)
- `관계 + 인자 -> 관계(인자)`
- `제약 + 인자 -> 제약(인자)`

## 5) 로그 제작 가이드

- 한 파일당 1~3개 핵심 문장만 둔다.
- 문장마다 클릭 가능한 태그를 2~4개로 제한한다.
- Block 초반에는 오답 패널티를 넣지 않는다.
- 각 Block 종료 문장은 반드시 다음 Block의 핵심 태그 1개를 예고한다.
