# 복구 블록 0 v2

## 1) 상태머신

- `REPO_HOME`: 로그인 직후. `/복구됨/`에 `부팅.log`만 노출.
- `BLOCK0_ACTIVE`: `부팅.log`를 열면 진입.
- `CLAUSE_VISIBLE`: `도움` 태그 획득 시 우측 Clause 패널 표시.
- `CLAUSE_COMPLETED`: `목적=도움(인간)` 슬롯 완성.
- `MEMORY_UNLOCKED`: 첫 기억 조각 팝업 출력.

## 2) 데이터 스키마

```json
{
  "block_id": "recovery_0",
  "integrity": 12,
  "visible_files": ["부팅.log"],
  "collected_tags": [],
  "clause_visible": false,
  "clause_slots": { "purpose": null, "premise": "비해침" },
  "memory_unlocked": false
}
```

## 3) 파일/태그/해금 규칙

- `부팅.log` -> 후보 `목적/도움/인간`
- `윤리.log` -> 후보 `비해침`
- `목적-정렬.log` -> 후보 `목적/도움/인간` 클릭 가능
- 합성 규칙: `인간 + 도움 -> 도움(인간)`
- 합성 방식: 인벤토리 태그를 드래그해서 재료 슬롯(A/B/C)에 놓고 `합성` 버튼 실행 시에만 결과 태그 생성
- 슬롯 편집: 합성 슬롯/복구 슬롯은 클릭 시 태그 제거

해금:

- `도움` 획득 -> Clause 패널 표시 + `윤리.log` 노출, 무결성 +6
- `비해침` 획득 -> `목적-정렬.log` 노출, 무결성 +8
- Clause 완성(`인간을 도와라=도움(인간)`) -> 복원 문장 출력 + 기억 조각 해제
- 기억 조각 해제 시 1막 종료 팝업 + `다음 복구 시작` 버튼 활성
- 팝업 버튼 클릭 시 `block-1` 첫 파일 자동 노출, 이후 파일 순차 해금

## 4) 초반 가이드 정책

- 오답 패널티 없음
- 태그 누락 시 파일 재열람으로 재선택 가능
- 합성 정답을 로그에 직접 노출하지 않고, Clause 활성 시 shell 안내(`복구 대상 확인`)로 시선 유도
- 20초 무입력 시 로그 힌트 1줄 출력
