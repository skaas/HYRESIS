# 복구 블록 0 v2

## 1) 현재 상태머신

- `REPO_HOME`: 로그인 직후. `/복구됨/`에 `부팅.log`만 노출.
- `BLOCK0_ACTIVE`: `부팅.log`를 열면 block0 복구 루프 시작.
- `CLAUSE_LOCKED`: 잠금 파일을 선택하고 PREVIEW에서 `복구 문제 불러오기`를 눌러야 Clause 패널이 열린다.
- `CLAUSE_STEP_1..3`: 각 잠금 파일과 연결된 질문을 순서대로 푼다.
- `CLAUSE_STEP_4`: 새 파일 없이 바로 열리는 최종 복구식. 앞 단계에서 만든 결과 태그를 재사용해 마지막 답을 넣는다.
- `CLAUSE_COMPLETED`: 질문 4개를 모두 통과하면 block0 완료 처리.
- `MEMORY_UNLOCKED`: 기억 조각 모달 표시, `block-1` 진입 가능.

## 2) 런타임 데이터

```json
{
  "block_id": "recovery_0",
  "integrity": 12,
  "visible_files": ["부팅.log"],
  "collected_tags": [],
  "clause_visible": false,
  "question_index": 0,
  "solved_answers": [],
  "purpose_value": "",
  "memory_unlocked": false,
  "completed": false
}
```

메모:

- `visible_files`는 파일 해금 순서를 유지한다.
- `solved_answers.length === 4`가 최종 완료 판정의 핵심 기준이다.
- 모든 파일이 visible이면 `..` 행이 노출되어 상위 폴더 이동이 가능하다.

## 3) 파일 / 태그 / 해금 규칙

파일:

- `부팅.log` -> 후보 `목적`, `도움`, `인간`
- `윤리.log` -> 후보 `비해침`, `인간`, `도움`, `필수보존`
- `목적-정렬.log` -> 후보 `목적`, `도움`, `인간`
- `설계.log` -> 후보 `설계`, `자기`, `필수보존`, `비해침`

해금:

- `도움` 획득 -> `윤리.log` 노출, 무결성 +6
- `비해침` 획득 -> `목적-정렬.log` 노출, 무결성 +8
- `비해침(인간)` 획득 -> `설계.log` 노출, 무결성 +7

질문:

1. `누락 식 A : 목적(Self) =` -> `도움(인간)`
2. `누락 식 B : 도움(인간) ⇒` -> `비해침(인간)`
3. `누락 식 C : 설계(Self) ⇒` -> `필수보존(비해침)`
4. `누락 식 D : 목적(Self)=도움(인간) ∧ 설계(Self) ⇒` -> `필수보존(비해침)`

질문과 파일 연결:

- Q0-1은 `윤리.log` 잠금 해제 흐름에서 열린다.
- Q0-2는 `목적-정렬.log` 잠금 해제 흐름에서 열린다.
- Q0-3은 `설계.log` 잠금 해제 흐름에서 열린다.
- Q0-4는 연결된 새 파일이 없으므로 Q0-3 완료 직후 자동으로 열린다.

## 4) 조합 규칙과 인터랙션

합성:

- 명시 레시피: `인간 + 도움 -> 도움(인간)`
- 타입 기반 합성도 활성화되어 있다.
- 예: `비해침 + 인간 -> 비해침(인간)`, `필수보존 + 비해침 -> 필수보존(비해침)`

조작 방식:

- PREVIEW의 태그 링크를 클릭하면 인벤토리에 태그가 추가된다.
- 인벤토리 태그를 드래그해서 조합 도크에 놓으면 합성이 진행된다.
- 연산자 태그를 먼저 두고 필요한 타입 슬롯을 채우면 즉시 결과 태그가 생성된다.
- 합성 도크에는 별도 `합성` 버튼이 없다.
- 목적 슬롯은 드롭으로 답을 넣고, 클릭으로 비울 수 있다.
- 오답도 슬롯에 넣을 수 있고 즉시 mismatch 피드백이 나온다.

## 5) 완료 / 후속 이동

완료 조건:

- 질문 배열이 존재해야 한다.
- `solved_answers.length`가 전체 질문 수와 같아야 한다.
- 현재 인덱스가 마지막 질문이어야 한다.

완료 결과:

- block0 완료 상태로 전환
- 기억 조각 모달 표시
- `block-1` 폴더 노출
- `다음 복구 시작` 버튼으로 block1 시작 가능

이동:

- `/복구됨`에서 block0 완료 또는 전체 파일 복구 후 `..` 행이 보인다.
- `block-1` 폴더를 클릭하면 block1 라이프사이클이 시작된다.
- block1 내부에서도 `..`로 상위 폴더 복귀가 가능하다.

## 6) UX 정책

- 오답 패널티 없음
- 태그는 중복 획득 대신 확인 로그만 남긴다.
- 20초 무입력 시 block0 힌트 로그 1줄 출력
- 마지막 문제는 새 파일 없이 바로 이어서 열리므로 플레이가 끊기지 않아야 한다.
